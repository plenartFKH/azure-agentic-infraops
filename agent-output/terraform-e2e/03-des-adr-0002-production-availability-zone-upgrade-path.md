## 🔍 Context

Reliability review identified that Standard S1 does not provide Availability Zone support. The architecture requires a valid production upgrade path that can increase resilience and still align with Azure App Service capabilities.

## ✅ Decision

Define **P1v4** as the production AZ-capable target for App Service when moving beyond dev-only reliability posture.

- Dev baseline remains App Service B1.
- Production uplift path is App Service Premium v4 (P1v4) with AZ support.
- Reliability uplift should be paired with SQL geo-replication and tested DR procedures.

## 🔄 Alternatives Considered

| Option | Pros | Cons | WAF Impact |
| ------ | ---- | ---- | ---------- |
| Stay on B1 indefinitely | Lowest cost | No autoscale, no AZ support | Cost +, Reliability - |
| Upgrade to S1 | Autoscale support | Still no AZ support | Performance +, Reliability - |
| Upgrade to P1v3 | AZ support available | Older premium generation | Reliability +, Cost - |
| Upgrade to P1v4 | AZ support + latest premium generation | Higher cost than B1/S1 | Reliability +, Cost - |

## ⚖️ Consequences

### Positive

- Removes ambiguity in production availability design
- Enables explicit reliability hardening milestone
- Aligns architecture docs with Azure platform capabilities

### Negative

- Material cost increase for production environments
- Requires readiness work (health checks, autoscale rules, load testing)

### Neutral

- No immediate change to dev deployment
- Decision can be revisited with actual traffic and SLO telemetry

## 🏛️ WAF Pillar Analysis

| Pillar | Impact | Notes |
| ------ | ------ | ----- |
| Security | Neutral | No direct security delta from tier change |
| Reliability | Strong Positive | AZ support and premium resiliency features |
| Performance | Positive | Improved compute headroom and scaling behavior |
| Cost Optimization | Negative | Higher baseline spend |
| Operational Excellence | Positive | Clear target state for runbooks and scaling policies |

## 🔒 Compliance Considerations

- No adverse impact on GDPR control mapping.
- Regional compliance constraints remain unchanged.

## 📝 Implementation Notes

- Apply this upgrade when production criteria are met (traffic/SLO gates).
- Pair with SQL geo-replication to avoid single-point regional failure.
- Recalculate monthly cost estimate in Step 4 plan before implementation.
